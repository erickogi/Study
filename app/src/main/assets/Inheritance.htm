------Inheritance is the mechanism by which an object acquires the some/all properties of another object.

 

-------It supports the concept of hierarchical classification.

 

For example: Car is a classification of Four Wheeler. Here Car acquires the properties of a four-wheeler. Other classifications could be a jeep, tempo, van etc. Four Wheeler defines a class of vehicles that have four wheels, and specific range of engine power, load carrying capacity etc. Car (termed as a sub-class) acquires these properties from Four Wheeler (termed as a super-class), and has some specific properties, which are different from other classifications of Four Wheeler, such as luxury, comfort, shape, size, usage etc.

 

 

A car can have further classification such as an open car, small car, big car etc, which will acquire the properties from both Four Wheeler and Car, but will still have some specific properties. This way the level of hierarchy can be extended to any level.

Inheritance

One of the most important relationships among objects in the real world is specialization, which can be described as the is-a-kind-of or is-a relationship.

 

When we say that a dog is a mammal, we mean that the dog is a specialized kind of mammal. It has all the characteristics of any mammal (it bears live young, nurses with milk, has hair), but it specializes these characteristics to the familiar characteristics of canis domesticus. A cat is also a mammal. As such, we expect it to share certain characteristics with the dog that are generalized in Mammal, but to differ in thosecharacteristics that are specialized in cats.

 

 

The ability of a new class to be created, from an existing class by extending it, is called inheritance.

 

In this case, the dog class and and the cat class each inherit the mammal class.

The specialization relationship is implemented using inheritance.

The specialization and generalization relationships are both reciprocal and hierarchical.

 

Specialization is just the other side of the generalization coin: Mammal generalizes whatis common between dogs and cats, and dogs and cats specialize mammals to their own specific subtypes.

Remember that abstraction reduces complexity by hiding irrelevant details.

 

Generalization

The concept of generalization occurs when several classes that already exist exhibit common functionality, structure, and purpose.

The analyst decides to create a new class to hold all the common attributes and functionality and make the existing classes extend it.

The new class is a generalized from the existing ones.

Specialization

The concept of Specialization occurs when you realize that the new class you are about to add has all the functionality, structure, and purpose of an existing class, but which also requires new behaviour or attributes or both. The new class is a specialized form of the existing one.

 

Abstract operations and classes

Abstract classes, which declared with the abstract keyword, cannot be instantiated.

 

It canonly be used as a super-class for other classes that extend the abstract class. Abstract class is the concept and implementation gets completed when it is being realized by a subclass.

In addition to this a class can inherit only from one abstract class (but a class may implement many interfaces) and and must override all its methods/properties that are declared to be abstract and may override virtual methods/ properties.

 

An abstract operation is an operation with no implementation. This is equivalent to a base class pure-virtual function in C++ or a deferred feature in Eiffel. In other words, it has an interface and a defined functionality but no implementation method with actual code.

 

For example, we could make getArea an abstract operation in Polygon, which would then become an abstract class. This has two implications:

----We would have to provide a concrete operation getArea for each of its subclasses.

---- We would never instantiate an object of class Polygon but a reference (or pointer) of type Polygon would still be used tohold the handle of an object of any of the subclasses.

 

Inheritance: reusing the interface

This is the mechanism by which attributes and the operations of a class become available to another class, as if those attributes and operations had been defined in the later class.

 

Class inheritance (usually referred to simply as inheritance) is an implementation of the generalization concept. Generalization states that the properties of an object type apply to its subtypes.

 

In most Object Oriented languages, the subclass inherits everything that the superclass has on offer; it doesn’t get to pick and choose what it inherits.

 

In single inheritance a class has exactly one direct superclass. Multiple inheritance is also possible. With multiple inheritance, each class has more than one direct superclasses.

 

 

Although inheritance may sometimes imply that you are going to add new functions to the interface, thats not necessarily true. The second and more important way to differentiate your new class is to change the behaviour of an existing base-class function.

 

This is referred to as overriding that function.

 

To override a function, you simply create a new definition for the function in the derived class. You are saying, I am using the same interface function here, but I want it to do something different for my new type.

 

Association

Association is a has-a or uses relationship between two classes where there is no particular ownership in place. It is just the connectivity between the two classes. All objects have their own lifecycle and there is no owner.

Lets take an example of Teacher and Student. They can exist without each other

The death of one party has no effect on the other party.

 

Aggregation

Aggregation is a specialized form of association. The addition is there that one of the parties is an owner and a child (the owned object) can not belong to another parent object. An example is a school Department and teacher. A teacher can only belong to one department.

 

Composition

Composition is a specialized form of aggregation. The addition is that neither party canexist meaningfully after the death of the other; if the parent object is deleted all child objects will also be deleted. Lets take again an example of relationship between House and rooms. House can contain multiple rooms there is no independent life of room and any room cannot belong to two different houses. If we delete the house - room will automatically be deleted

 

-- Some authors refer to aggregation as weak association and composition as strong association.

--Some authors refer to aggregation as strong association and composition as strong aggregation.

--Some authors refer to both aggregation and composition as strong type of association but distinguish them as (with partial ownership) and (with full ownership) respectively.

--Some authors refer to composition as the (death) relationship if the lifetime of the objects are the same.

 

Degree of an Association

Degree of an association denotes the number of classes involved in a connection. Degree may be unary, binary, or ternary.

----A unary relationship connects objects of the same class.

----A binary relationship connects objects of two classes.

--- A ternary relationship connects objects of more than two classes.

Cardinality Ratios of Associations

Cardinality of a binary association denotes the number of instances participating in anassociation. There are three types of cardinality ratios, namely:

---One--to--One : A single object of class A is associated with a single object of class B.

---One--to--Many : A single object of class A is associated with many objects of class B.

---Many--to--Many : An object of class A may be associated with many objects of class B and conversely an object of class B may be associated with many objects of class

 

Composition: Reusing the implementation

Composition is often referred to as a has-a relationship, as in a car has an engine.

 

Composition comes with a great deal of flexibility. The member objects of your new class are usually private, making them inaccessible to the client programmers who are using the class. This allows you to change those members without disturbing existing client code. You can also change the member objects at runtime, to dynamically change the behaviour of your program.

 

Propagation and Delegation

In Object-oriented programming, Association defines a relationship between classes of objects which allows one object instance to cause another to perform an action on its behalf. This relationship is structural, because it specifies that objects of one kind are connected to objects of another.

In generic terms the causation is usually called "sending a message", "invoking a method" or "calling a member function" to the controlled object. Concrete implementation usually requires the requesting object to invoke a method or member function using a reference or pointer to the memory location of the controlled object.

 

Propagation

Propagation is a principle that you can apply to classes that are associated with others through both aggregation and composition.

 

If a system requires a Car object, it simply instantiates the Car class. The CarRadio, Engine, and other associated objects are created by the Car. Therefore, the Car class acts as an interface to the others, possibly without the owner of the Car even knowing they exist.

 

In such systems, the owner of the Car can invoke methods on that object. These methods can in turn invoke methods on any of the objects of which it is an aggregate, or from which it is composed. Consider the Car class having a pushAccelerator() method.

 

When the system invokes the pushAccelerator() method of the Car object, it is attempting to alter something about the Car (in this case, its speed). The internal state of the Car class must change, but so does the state of some of the associated objects.

 

This means that invoking the Car’s pushAccelerator() method involves the invocation of one or more methods on the associated objects. In this case, one of the objects affected by the pushAccelerate() method is the Engine. The Engine has an appropriate method for the Car to invoke, possibly called accelerate().

 

The actions of the pushAccelerator() method involves invoking the accelerate() method. This should not be the whole task. The invocation of one method, causing one or more methods on associated objects to be invoked in this way, is called Propagation.

 

Delegation

Delegation is another principle that applies to classes that are related through association. Propagation involved invoking a method of the Car class to invoke a method of another class. Delegation is where the entire operation that the user or system invokes on the

Car class actually takes place within associated objects.

 

For example, the Car class has a turnMusicDown() method. This is an operation that the user associates with the Car, because that class is the only class to which the user has a reference. But because the analysis has created a separate CarRadio class, which is  actually responsible for the volume of the music within the Car, the Car delegates that  responsibility.

 

Both the Car and CarRadio classes have a turnMusicDown() method. In the CarRadio class, it actually alters the volume attribute. In the Car class, it simply invokes the turnMusicDown() method of the CarRadio.

 

 

 

The process by which one class acquires the properties and functionalities of another class. Inheritance provides the idea of reusability of code and each sub class defines only those features that are unique to it.

class. The new class which inherits from the base class is called the derived class or sub class or child class.

 

4.    Inheritance implements the ―Is-A‖ or ―Kind Of/ Has-A‖ relationship.

 

Note : The biggest advantage of Inheritance is that, code in base class need not be rewritten in the derived class.

 

The member variables and methods of the base class can be used in the derived class as well.

Types of Inheritance

Multilevel Inheritance

 

Multilevel inheritance refers to a mechanism in OO technology where one can inherit from a derived class, thereby making this derived class the base class for the new class.

 

Multiple Inheritance

―Multiple Inheritance‖ refers to the concept of one class inheriting from more than

 

one base class. The inheritance we learnt earlier had the concept of one base class or

 

parent. The problem with ―multiple inheritance‖ is that the derived class will have to

 

manage the dependency on two base classes.

 

Note 1: Multiple Inheritance is very rarely used in software projects. Using Multiple inheritance often leads to problems in the hierarchy. This results in unwanted complexity when further extending the class.

 

Note 2: Most of the new OO languages like Small Talk, Java, C# do not support Multiple inheritance. Multiple Inheritance is supported in C++.

1) Single Inheritance

Single inheritance is damn easy to understand. When a class extends another one class only then we  call it a single inheritance. The below flow diagram shows that class B extends only one class which is A. Here A is a parent class of B and B would be  a child class of A.

Single Inheritance example program in Java

Class A

{

   public void methodA()

   {

     System.out.println("Base class method");

   }

}

 

Class B extends A

{

   public void methodB()

   {

     System.out.println("Child class method");

   }

   public static void main(String args[])

   {

     B obj = new B();

     obj.methodA(); //calling super class method

     obj.methodB(); //calling local method

  }

}

2) Multiple Inheritance

“Multiple Inheritance” refers to the concept of one class extending (Or inherits) more than one base class. The inheritance we learnt earlier had the concept of one base class or parent. The problem with “multiple inheritance” is that the derived class will have to manage the dependency on two base classes.

Note 1: Multiple Inheritance is very rarely used in software projects. Using Multiple inheritance often leads to problems in the hierarchy. This results in unwanted complexity when further extending the class.

Note 2: Most of the new OO languages like Small Talk, Java, C# do not support Multiple inheritance. Multiple Inheritance is supported in C++.

3) Multilevel Inheritance

Multilevel inheritance refers to a mechanism in OO technology where one can inherit from a derived class, thereby making this derived class the base class for the new class.
