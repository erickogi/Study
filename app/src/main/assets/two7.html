PROJECT PLANNING

1 Effective management of software project depends on thoroughly planning the progress of the project.

2 SPM’s draw a plan to be used as the driver for the project - organizational goals a must

3 Planning an iterative process – only complete when the project itself is complete.

4 Assess the constraints affecting the project (required delivery date, staff available, overall budget, etc).

5 Estimate project parameters (structure, size, distribution of functions).

6 Define progress milestones and deliverables.

7 Schedule Project Technical Review – renegotiate milestones and constraints.

Milestones vs. Deliverables

Milestones: A predictable outcome of an activity where some formal report of progress should be presented to management.

Deliverables: is a project result that is delivered to the customer.
Delivered at the end of some major project phase eg. specification or design. Deliverables are usually milestones, but milestones need not be deliverables.
